Decision Tree

Feature Engineering

In total, 10 features are created for each input signal (command control current and the feedback signal sensor). To this end, simple signal processing techniques are employed, such as delay, differences, moving average, moving standard deviation, moving root mean squared (RMS) value, and zero-cross counting. The selection of such features takes into account the limited computational resources available in FCSs.

Data acquisiton

Data acquisition is necessary for generating inputs and outputs for training and validating the DT. To this end, we develop a loop routine to simulate multiple scenarios combining different parameters for the OFC and control actions. Each run is composed of 60 seconds, or 2, 401 observations. As a result, 82, 297, 856 observations are captured in total. To limit computational burden, only 20% of the data is randomly selected for training and validation.

Model Training

To develop the OFC detection model, a DT is trained using a random sample with 50% of the selected data. The algorithm is configured with minimal number of leaf node observations equals to 1, minimum number of branch node observations equals to 10, and Gini’s diversity index as the split criterion Breiman (2017).

Model Validation

This works use real-word data, from the major german water supplier, generated in a research project on drinking water.To validate the resulting DT, the remaining 50% of the selected data is employed. Additionally, tree pruning is performed to improve generalization by removing model’s branches Breiman (2017), with prune ratios ranging from 0% to 99%.
